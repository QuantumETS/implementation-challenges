# Toy QeMCMC for Maximum Independent Set

## Abstract

This assignment consists of implementing a simplified quantum-enhanced Markov Chain Monte Carlo workflow for the Maximum Independent Set problem. The challenge is inspired by the IBM Quantum publication **“Quantum-enhanced Markov Chain Monte Carlo for Combinatorial Optimization”**.

The original work investigates quantum-enhanced MCMC methods for combinatorial optimization, including Maximum Independent Set instances, using techniques such as warm-starting and parallel tempering. This assignment does not attempt to reproduce the scale, hardware execution, or performance claims of the paper. Instead, it reproduces the main algorithmic structure in a toy setting using small graph instances and Qiskit simulators.

The implementation targets two Maximum Independent Set instances: one with 5 binary decision variables and one with 8 binary decision variables. The 5-variable model is intended for debugging, exhaustive verification, and visualization of the full solution landscape. The 8-variable model is intended to demonstrate comparative behavior between classical and quantum-enhanced proposal strategies while remaining small enough for exact brute-force validation.

The expected workflow is to encode the Maximum Independent Set problem as a binary optimization model, verify the exact optimum by exhaustive search, implement a classical Metropolis MCMC baseline, and then replace the classical local proposal rule with a Qiskit-based quantum proposal circuit. The quantum circuit acts as a proposal generator inside the MCMC loop, while the accept/reject step remains classical.

The assignment should produce a small experimental study comparing random-start classical MCMC, warm-start classical MCMC, and a toy QeMCMC-inspired sampler.

## Mathematical Model

Given an undirected graph

$$
G = (V, E),
$$

where $V$ is the set of vertices and $E$ is the set of edges, the Maximum Independent Set problem consists of finding the largest subset of vertices such that no two selected vertices are adjacent.

Each vertex $i \in V$ is represented by a binary decision variable:

$$
x_i =
\begin{cases}
1, & \text{if vertex } i \text{ is selected}, \\
0, & \text{otherwise}.
\end{cases}
$$

A candidate solution is therefore represented by a bitstring

$$
x = (x_0, x_1, \dots, x_{n-1}) \in \{0,1\}^n,
$$

where $n = |V|$.

The size of the selected vertex set is

$$
\sum_i x_i.
$$

The independent-set constraint requires that no edge has both endpoints selected. For every edge $(i,j) \in E$, the following condition should hold:

$$
x_i x_j = 0.
$$

A convenient unconstrained minimization objective is obtained by rewarding selected vertices and penalizing invalid edges:

$$
C(x) = -\sum_i x_i + \lambda \sum_{(i,j) \in E} x_i x_j.
$$

The first term rewards larger selected sets. The second term penalizes configurations that select adjacent vertices. The penalty coefficient $\lambda$ must be large enough to make invalid solutions less favorable than valid independent sets of comparable size. For the toy instances in this assignment, values such as

$$
\lambda = 2
$$

or

$$
\lambda = |V|
$$

are reasonable starting points.

The global optimum should be verified exactly by exhaustive enumeration. This is feasible because the two required problem sizes contain only

$$
2^5 = 32
$$

and

$$
2^8 = 256
$$

possible bitstrings.

The classical Metropolis acceptance rule is defined using the energy difference

$$
\Delta C = C(x') - C(x),
$$

where $x$ is the current state and $x'$ is the proposed state. The proposal is accepted with probability

$$
P_{\mathrm{accept}} = \min\left(1, e^{-\beta \Delta C}\right),
$$

where $\beta$ is the inverse temperature. Lower values of $\beta$ allow broader exploration, while higher values of $\beta$ make the chain greedier with respect to the objective function.

In the toy QeMCMC-inspired version, the proposal $x'$ is generated by a Qiskit circuit rather than by a single classical bit flip. A suggested proposal circuit may follow this structure:

1. Prepare a quantum register initialized from, or biased toward, the current bitstring $x$.
2. Apply a layer of single-qubit mixing gates, such as $R_X(\theta)$ rotations.
3. Optionally apply cost-inspired entangling gates associated with the graph edges.
4. Measure all qubits in the computational basis.
5. Use the measured bitstring as the proposed state $x'$.
6. Accept or reject $x'$ using the classical Metropolis acceptance rule.

The quantum circuit is not expected to solve the optimization problem directly. It acts as a stochastic proposal mechanism inside a classical MCMC loop.

## Milestones

### Milestone 1: Maximum Independent Set Encoding and Brute-Force Verification

Define one 5-node graph and one 8-node graph. The graphs may be manually specified or generated from fixed random seeds to ensure reproducibility.

Implement the binary encoding of candidate independent sets. The implementation should include functions to:

- convert between bitstrings and selected vertex sets;
- evaluate the cost function $C(x)$;
- determine whether a bitstring represents a valid independent set;
- compute the size of a selected vertex set;
- enumerate all possible bitstrings;
- identify the exact optimal independent set by brute force.

This milestone establishes the ground truth used to validate all sampling-based methods.

### Milestone 2: Classical Metropolis MCMC Baseline

Implement a classical Metropolis MCMC sampler for the Maximum Independent Set objective.

A simple proposal rule is sufficient: at each iteration, select one bit uniformly at random and flip it. The resulting bitstring becomes the proposed state. The proposal is then accepted or rejected according to the Metropolis acceptance rule.

The implementation should record the current state, current cost, best state found so far, best cost found so far, validity of the current state, and size of the selected set. The sampler should be executed on both the 5-node and 8-node graph instances.

This milestone provides a classical baseline for comparison with the QeMCMC-inspired proposal mechanism.

### Milestone 3: Warm-Started Classical MCMC

Implement a warm-start strategy for the classical MCMC sampler.

A simple greedy heuristic may be used to generate an initial independent set. For example, vertices may be considered in increasing degree order, adding a vertex only if it does not conflict with vertices already selected. Other deterministic or randomized greedy heuristics are also acceptable if they are clearly documented.

Run the same classical MCMC algorithm twice: once from a random initial bitstring and once from the warm-start bitstring. Compare convergence behavior and final solution quality.

This milestone reproduces, in simplified form, the idea that initialization can affect the performance of stochastic optimization methods.

### Milestone 4: Qiskit-Based Quantum Proposal Kernel

Implement a Qiskit-based proposal circuit that generates candidate bitstrings for the MCMC loop.

The quantum proposal circuit should contain one qubit per binary decision variable. For the required instances, this corresponds to 5-qubit and 8-qubit circuits. The circuit should be executed using a Qiskit simulator.

A minimal implementation may use the following pattern:

- initialize the circuit according to the current bitstring;
- apply parameterized mixing gates such as $R_X(\theta)$;
- optionally apply graph-aware entangling gates or phase gates associated with edges in $E$;
- measure the circuit;
- use the measured bitstring as the proposed next state.

The proposed bitstring should then be accepted or rejected using the same Metropolis rule used in the classical MCMC sampler. This keeps the comparison focused on the proposal mechanism while preserving the same objective and acceptance dynamics.

The QeMCMC-inspired sampler should be executed on both the 5-node and 8-node graph instances.

### Milestone 5: Experimental Comparison

Compare at least three methods:

| Method | Description |
|---|---|
| Random-start classical MCMC | Classical local bit-flip proposal initialized from a random bitstring |
| Warm-start classical MCMC | Classical local bit-flip proposal initialized from a greedy independent set |
| Toy QeMCMC | Qiskit-based quantum proposal circuit inside the MCMC loop |

Each method should be run multiple times to account for stochastic variation. The number of repeated runs does not need to be large, but it should be sufficient to compare convergence and final solution quality. For example, 20 to 50 repetitions per method is suitable for a toy study.

The comparison should be performed separately for the 5-node and 8-node graph instances. The exact optimum obtained by brute force should be used as the reference solution.

The analysis should avoid claims of quantum advantage. The objective is to reproduce the structure of a quantum-enhanced MCMC workflow at toy scale, not to demonstrate asymptotic or practical superiority.
